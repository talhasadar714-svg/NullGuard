# NullGuard / NullShield

An enterprise-grade, zero-overhead compile-time null safety enforcer for C# using Roslyn Incremental Source Generators.

NullShield analyzes your methods decorated with the `[NullShield]` attribute and automatically generates high-performance, strictly typed guard code to protect your logic against null reference exceptions.

## Installation & Setup

1. Add **NullShield.Core** to your project to get access to the attributes and enums.
2. Add **NullShield.Generators** as an Analyzer/Source Generator in your project:
   ```xml
   <ItemGroup>
       <ProjectReference Include="..\NullShield.Generators\NullShield.Generators.csproj" OutputItemType="Analyzer" ReferenceOutputAssembly="false" />
   </ItemGroup>
   ```

## Getting Started

To use NullShield, simply attach the `[NullShield]` attribute to any method whose parameters you wish to protect.

```csharp
using System;
using NullShield.Core.Attributes;
using NullShield.Core.Enums;

namespace MyApplication
{
    public class UserService
    {
        // 1. Attach the attribute to your method
        [NullShield(MitigationStrategy.ThrowException)]
        public static void SaveUser(string username, string email)
        {
            // 2. Call the auto-generated guard method at the top of your function
            NullShield_Guard_UserService_SaveUser.ValidateParameters(username, email);

            Console.WriteLine($"Processing user: {username}");
        }
    }
}
```

### How It Works

During the build process, the NullShield Roslyn Source Generator intercepts the `[NullShield]` attribute and auto-generates a static class named `NullShield_Guard_{ClassName}_{MethodName}`.

In the example above, the generator emits the following class:
```csharp
// <auto-generated/>
using System;

namespace MyApplication
{
    public static class NullShield_Guard_UserService_SaveUser
    {
        public static void ValidateParameters(string username, string email)
        {
            if (username == null) throw new ArgumentNullException(nameof(username));
            if (email == null) throw new ArgumentNullException(nameof(email));
        }
    }
}
```

## Mitigation Strategies

NullShield gives you fine-grained control over how null parameters are handled via the `MitigationStrategy` enum. 

- **`ThrowException` (Default)**: Instantly throws an `ArgumentNullException` on null values. Preferred for strict contracts and enterprise environments.
- **`FallbackToDefault`**: *(Coming Soon)* Assigns a safe, non-null default value instead of failing.
- **`LogAndBypass`**: *(Coming Soon)* Gently logs the issue to the diagnostics/console and bypasses the execution safely without crashing the application.

## Best Practices

- Always place the call to `ValidateParameters` as the very first line inside your decorated method.
- Since the source generator relies on standard Roslyn integration, any IDE (such as Visual Studio, JetBrains Rider, or VS Code) will instantly recognize the generated `NullShield_Guard_*` classes with full IntelliSense support.
